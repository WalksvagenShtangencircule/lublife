## Safe update and scaling checklist

Этот сценарий закрывает типовые потери при обновлении из репозитория: кастомные мобильные меню, локальные расширения, конфиги и runtime-правки.

### 1) Подготовка

```bash
cd /opt/rbt
git status --short
```

Если есть незакоммиченные изменения в коде (кроме конфигов/данных), сначала зафиксируйте их в вашей ветке.

### 2) Безопасное обновление

```bash
cd /opt/rbt
./install/scripts/safe_update.sh main
```

Что делает скрипт:

- сохраняет резервную копию критичных локальных путей в `/opt/backups/smartaccess/<timestamp>`;
- запускает `php server/cli.php --backup-db`;
- обновляет git-ветку от `origin`;
- запускает `composer install`;
- возвращает сохранённые конфиги/кастомизации;
- выполняет `--init-db`, `--init-clickhouse-db`, `--clear-cache`, `--reindex`.

### 3) Что хранить в репозитории обязательно

Эти файлы влияют на мобильные дополнительные пункты меню и должны быть в git:

- `server/mobile/ext/custom/ext.php`
- `server/mobile/ext/custom/list.php`
- `server/mobile/ext/custom/options.php`
- `server/mobile/address/custom/offices.php`
- `client/modules/custom/custom.js`
- `client/modules/custom/i18n/*.json`
- `static/portal/keys/**`

### 4) Проверка после обновления

- открыть мобильное приложение и убедиться, что видны дополнительные пункты меню;
- проверить выдачу `/mobile/ext/list` и `/mobile/ext/ext`;
- проверить `server/config/config.json` и `client/config/config.json`;
- убедиться, что фоновые сервисы и cron работают штатно.

### 5) Перенос на новый сервер (масштабирование)

- разворачивайте код из этого репозитория;
- переносите БД и Mongo отдельно (см. `server/db/migration/README.md`);
- затем запускайте `safe_update.sh` для унификации конфигов, схемы и индексов.
