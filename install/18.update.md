# Обновление SmartAccess/SmartYard

Ниже описан безопасный процесс обновления для форка (`SmartAccess`) и для upstream (`SmartYard-Server`).

## Быстрое обновление (рекомендуется)

```bash
sudo ./install/scripts/rbt-deploy.sh update \
  --target /opt/rbt \
  --ref main
```

Для обновления до конкретной ветки:

```bash
sudo ./install/scripts/rbt-deploy.sh update \
  --target /opt/rbt \
  --ref chore/upstream-safe-docs-sync
```

## Перед обновлением: проверьте локальные изменения

```bash
cd /opt/rbt
git status
git stash push -u -m "pre-update-$(date +%F-%H%M)"
```

Если есть собственные патчи, сохраните их:

```bash
cd /opt/rbt
git format-patch -1 HEAD
```

## Сравнение форка с upstream SmartYard

```bash
cd /opt/rbt
git remote -v
git fetch upstream --tags
git diff --name-status upstream/main HEAD
```

Если история репозитория переписана и нет общего merge-base, используйте прямое сравнение деревьев:

```bash
cd /opt/rbt
git diff --name-status upstream-main HEAD
```

## Ручное обновление (fallback)

```bash
cd /opt/rbt
git fetch --all --tags --prune
git checkout main
git pull --ff-only origin main
cd /opt/rbt/server && composer install --no-interaction
cd /opt/rbt/client/lib/Leaflet && npm install && npm run build
php /opt/rbt/server/cli.php --strip-config || true
```

## После обновления

```bash
systemctl restart php8.3-fpm || true
systemctl restart nginx || true
systemctl restart pgbouncer || true
systemctl restart clickhouse-server || true
```

Проверка:

```bash
php /opt/rbt/server/cli.php --init-db
php /opt/rbt/server/cli.php --reindex
```

Если у вас кастомные совместимые патчи (пример: `install/patches/*.patch`), применяйте их только на отдельной ветке и проверяйте `git apply --check` до применения.
