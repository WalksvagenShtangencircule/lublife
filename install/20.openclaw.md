# OpenClaw Gateway

OpenClaw — локальный AI-агент, обеспечивает MCP-инструменты для модуля ИИ-ассистента. В данной конфигурации используется как MCP-провайдер для mcp-assistant.

## Требования

- Node.js >= 22.x
- pnpm >= 9.x
- Пользователь `tech` (или тот, под которым работает RBT)

## Установка Node.js 22

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

## Установка pnpm

```bash
sudo npm install -g pnpm
```

## Установка OpenClaw

```bash
sudo git clone https://github.com/openclaw/openclaw.git /opt/openclaw
sudo chown -R tech:tech /opt/openclaw
cd /opt/openclaw
PATH=/usr/local/bin:/usr/bin:/bin pnpm install
PATH=/usr/local/bin:/usr/bin:/bin pnpm build
```

## Конфигурация

Создать файл `~/.openclaw/openclaw.json`:

```json
{
  "agents": {
    "defaults": {
      "model": "deepseek/deepseek-v4-flash",
      "agentRuntime": {
        "id": "pi",
        "fallback": "none"
      },
      "timeoutSeconds": 120
    }
  },
  "gateway": {
    "mode": "local",
    "port": 18789,
    "bind": "loopback",
    "auth": {
      "token": "ЗАМЕНИТЕ_НА_СЛУЧАЙНЫЙ_ТОКЕН",
      "mode": "token"
    },
    "http": {
      "endpoints": {
        "chatCompletions": {
          "enabled": true
        }
      }
    }
  },
  "mcp": {
    "servers": {
      "smartaccess": {
        "command": "/usr/bin/node",
        "args": ["/opt/rbt/server/services/mcp-assistant/index.mjs"],
        "env": {
          "SMARTACCESS_BASE_URL": "https://127.0.0.1/api",
          "SMARTACCESS_TOKEN": "ТОКЕН_АДМИНИСТРАТОРА_ИЗ_REDIS",
          "SMARTACCESS_MCP_HTTP_METHODS": "GET,POST"
        }
      }
    }
  },
  "plugins": {
    "entries": {
      "deepseek": {
        "enabled": true
      }
    }
  }
}
```

Создать файл `~/.openclaw/agents/main/agent/auth-profiles.json`:

```json
{
  "deepseek": {
    "id": "deepseek",
    "label": "DeepSeek",
    "providerId": "deepseek",
    "authType": "api_key",
    "key": "DEEPSEEK_API_KEY"
  }
}
```

## Запуск как systemd-сервис

```bash
sudo cp /opt/rbt/install/systemd/openclaw-gateway.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable openclaw-gateway
sudo systemctl start openclaw-gateway
sudo systemctl status openclaw-gateway
```

## Проверка

```bash
curl -s http://127.0.0.1:18789/v1/models \
  -H "Authorization: Bearer ЗАМЕНИТЕ_НА_СЛУЧАЙНЫЙ_ТОКЕН"
```

## Настройка ассистента в config.json

В секции `assistant` файла `server/config/config.json` (см. пример `server/config/assistant.example.json`):

```json
"assistant": {
  "deepseekApiKey": "DEEPSEEK_API_KEY",
  "deepseekBaseUrl": "https://api.deepseek.com",
  "model": "deepseek-v4-flash",
  "maxToolIterations": 6,
  "maxToolCallsPerRequest": 3,
  "maxResponseTokens": 520,
  "maxTotalSeconds": 90
}
```

## Права доступа в PostgreSQL

Пользователю с ролью Administrator необходимы права на методы `assistant/chat`, `diagnostics/check`, `diagnostics/action`. Права выдаются через таблицу `core_users_rights`:

```sql
INSERT INTO core_users_rights (uid, rights)
VALUES (UID_АДМИНИСТРАТОРА, '{}')
ON CONFLICT (uid) DO NOTHING;

UPDATE core_users_rights
SET rights = rights
  || jsonb_build_object('12c6ba4473c55d57663a71ff4a953bec', true)  -- assistant/chat
  || jsonb_build_object('f014b55f08bf8ba783bffd58de03b86e', true)  -- diagnostics/check
  || jsonb_build_object('d8a50deb62e2e54ab4b6a61cf214cd97', true)  -- diagnostics/action
  || jsonb_build_object('3000443aa69012bf3572e97f54761ced', true)  -- diagnostics/run
WHERE uid = UID_АДМИНИСТРАТОРА;
```
